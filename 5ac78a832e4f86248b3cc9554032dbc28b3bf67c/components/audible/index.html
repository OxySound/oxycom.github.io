    Protocols¶

    OxySound SDKs can be configured to use different communication protocols, which determine various properties of the transmission.

    All¶

    OxySound will listen simultaneously for multiple protocols, for example sending an audible then an Near-ultrasonic signal will both be processed without restarting the SDK unless a specific mode is selected or custom frequencies are in use. The default is All

    Audible¶

    Audible frequencies are recommended for channels which have a limited audio sample rate -- VoIP connections, lossy codecs, or lower-spec embedded devices. Specific protocols are available for each of these scenarios.

    Near-ultrasonic¶

    Near-ultrasonic frequencies should be used when noise disturbance is not wanted. Frequencies between 17kHz and 21kHz are supported by virtually all mobiles and computers, but are generally inaudible to the human ear.

    How it will sound¶

    For scenarios in which a more tuneful tone is desirable -- for example, in children's toys -- the transmission can be tailored to sound more tuneful. By using ultrasonic data layered over another piece of sound, devices can even appear to communicate using arbitrary audio.

    Compression¶

    For scenarios where embeding into media is desirable -- for example, in Youtube -- the transmission can be tailored to send second screen experience. By using data layered over another is possible to send more than data transmission for example the seconds of the video and a custom message.

    Custom Frequencies¶

    For scenarios where control is a must you can choose a start and end frequency.

    Additional protocols¶

    Available on request, including:

    1. pstn: for transmission over phone lines and VoIP
    2. long-range: for large venues, industrial and stadium transmissions
    3. nfc: for high-speed, short-range transmission
